# ModelChain - Complete Use Case Documentation
## Decentralized AI Model Marketplace

---

## PROJECT OVERVIEW
ModelChain is a blockchain-based marketplace where developers can upload, license, and sell AI models as NFTs. The platform uses smart contracts for transparent ownership, trustless payments, and decentralized governance.

**Technology Stack:**
- Blockchain: Polygon Amoy Testnet
- Storage: IPFS (Pinata)
- Frontend: React + Vite
- Smart Contracts: Solidity (Hardhat)
- Authentication: Wallet-only (MetaMask, WalletConnect)

---

## ACTORS

### 1. VISITOR (Unauthenticated)
- Anyone browsing the platform without wallet connection
- Limited access to public content only

### 2. USER (Wallet Connected)
- Basic authenticated user with wallet connected
- Can purchase models, test in sandbox, view profile
- Inherits: Visitor capabilities

### 3. DEVELOPER (Model Creator)
- Specialized user who creates and uploads AI models
- Can earn from model sales and track analytics
- Inherits: User capabilities

### 4. VALIDATOR (Quality Checker)
- Specialized user who validates model quality
- Earns rewards for accurate validations
- Inherits: User capabilities

### 5. DAO MEMBER (Token Holder)
- User with governance tokens (MCT)
- Can create and vote on platform governance proposals
- Inherits: User capabilities

### 6. ADMIN (Platform Manager)
- Platform administrator with full management access
- Can moderate content, manage users, emergency controls
- Inherits: User capabilities

### 7. EXTERNAL SYSTEMS
- MetaMask/WalletConnect (Wallet providers)
- IPFS/Pinata (Decentralized storage)
- Blockchain (Smart contract execution)

## 1. AUTHENTICATION & WALLET MANAGEMENT (10 Use Cases)

### UC1: Connect Wallet
**Actor:** Visitor, User
**Description:** User connects their Web3 wallet (MetaMask/WalletConnect) to authenticate
**Preconditions:** User has a Web3 wallet installed
**Flow:**
1. User clicks "Connect Wallet" button
2. System displays wallet provider selection modal
3. User selects wallet provider (MetaMask/WalletConnect)
4. Wallet prompts for connection approval
5. User approves connection
6. System verifies wallet signature with blockchain
7. System checks if user is registered in UserRegistry contract
8. If not registered: System displays onboarding modal (UC3)
9. If registered: System loads user profile from IPFS
10. System authenticates user and redirects to dashboard
**Postconditions:** User is authenticated with wallet connected
**Extensions:** 
- If wallet not installed: Show installation guide
- If wrong network: Prompt to switch to Polygon Amoy

### UC2: Disconnect Wallet
**Actor:** User
**Description:** User disconnects their wallet and logs out
**Preconditions:** User is connected
**Flow:**
1. User clicks disconnect/sign out
2. System calls wallet disconnect
3. System clears authentication state
4. System redirects to connect wallet page
**Postconditions:** User is logged out, wallet disconnected

### UC3: Complete Onboarding
**Actor:** New User
**Description:** First-time user completes profile creation
**Preconditions:** Wallet connected, not registered
**Flow:**
1. System detects new user (not in UserRegistry)
2. System displays onboarding modal
3. User enters display name, email (optional)
4. User selects primary role (UC4)
5. User uploads avatar (optional)
6. User confirms profile data
7. System uploads profile to IPFS via Pinata
8. System calls UserRegistry.registerUser() on blockchain
9. Transaction is mined on Polygon Amoy
10. System stores IPFS hash in smart contract
11. System displays success message
12. System redirects based on role
**Postconditions:** User profile created on blockchain and IPFS

### UC4: Select User Role
**Actor:** New User
**Description:** User selects their primary role during onboarding
**Preconditions:** Onboarding in progress
**Options:**
- Buyer (Role 0) - Purchase and use models
- Developer (Role 1) - Upload and sell models
- Validator (Role 2) - Validate model quality
**Postconditions:** Role stored in UserRegistry contract

### UC5: Create Profile
**Actor:** New User
**Description:** User creates blockchain profile
**Included in:** UC3
**Smart Contract:** UserRegistry.registerUser()

### UC6: View Wallet Balance
**Actor:** User
**Description:** User views their POL/ETH balance
**Preconditions:** Wallet connected
**Display:** Shows balance in native currency (POL for Polygon)

### UC7: View Transaction History
**Actor:** User
**Description:** User views their blockchain transaction history
**Display:** Empty state (real data from blockchain coming soon)

### UC8: Switch Network
**Actor:** User
**Description:** User switches blockchain network
**Flow:**
1. System detects current network from wallet
2. If not Polygon Amoy: Show warning
3. User clicks switch network
4. Wallet prompts for network change
5. System updates UI with new network info

### UC9: Copy Wallet Address
**Actor:** User
**Description:** User copies their wallet address to clipboard
**Flow:**

### UC10: View on Block Explorer
